Examples

Non-Extensible Developments

  • Non-extensible: An introduction to basic reasoning in Extensibella, proving properties about a few list relations without extensibility

Lambda Calculi

  • Walkthrough language: Simply-typed lambda calculus with syntactic additions
  • Simply-typed lambda calculus: Simply-typed lambda calculus showing preservation and progress with syntactic additions
  • Lambda calculus: The untyped lambda calculus as a host language with typing added as an extension

Imperative Languages

  • Paper library: Imperative language with statements and expressions with syntactic and semantic additions as seen in "A Modular Approach to Metatheoretic Reasoning for Extensible Languages"
  • Simple imperative language: Imperative language with statements and expressions, expanding on the paper library example, with syntactic and semantic additions
  • Large imperative language with loose constraints: Imperative language with expressions, statements, and functions that declares loose projection constraints
  • Large imperative language with evaluation through matching: Imperative language with expressions, statements, and functions that defines evaluation using matching through projection instead of exact forms with middling projection constraints
  • Large imperative language with strict constraints: Imperative language with expressions, statements, and functions that declares strict projection constraints

Other Languages

  • UNSAT proofs: Language for proving CNF (Boolean satisfiability) problems are unsatisfiable with syntactic and semantic additions

Modeling Other Modular Proof Frameworks

  • SoundX: SoundX shows extension-introduced typing desugars to host-language-introduced typing
  • Meta-Theory a la Carte: MTC allows extensions building on a shared base of declarations to prove shared properties
